## 폴링과 웹 훅에 대해서 이해해보자

폴링(Polling)은 클라이언트가 일정한 시간 간격으로 서버에 요청을 전송하여 상태 변화를 확인하는 방식입니다.
이 방식은 변경 사항이 없는 경우에도 반복적인 요청과 응답이 발생하여 네트워크 및 서버 자원이 비효율적으로 사용될 수 있습니다.
또한 폴링 주기에 의존하기 때문에 실시간 처리에 한계가 존재합니다.

웹 훅(Webhook)은 특정 이벤트가 발생했을 때 서버가 사전에 등록된 클라이언트의 URL로 HTTP 요청을 전송하여 이를 통지하는 방식입니다.
클라이언트는 주기적인 요청을 수행할 필요가 없으며, 이벤트 발생 즉시 알림을 수신할 수 있습니다.
이로 인해 웹 훅은 네트워크 부하를 줄이면서 높은 실시간성을 제공할 수 있습니다.

결론적으로, 폴링은 구현이 단순하다는 장점이 있으나 효율성과 실시간성 측면에서 한계가 있습니다.
반면, 웹 훅은 초기 설정이 필요하지만 이벤트 기반으로 동작하여 실시간 이벤트 처리가 요구되는 시스템에 보다 적합한 방식입니다.
