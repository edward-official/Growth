## 1. 프로젝트 개요
본 프로젝트는 AI 에이전트와 RAG 파이프라인을 노드 기반 시각적 인터페이스로 설계하고, 이를 실제 실행 가능한 AI 워크플로우로 변환·실행·관찰할 수 있는 플랫폼을 구축하는 것을 목표로 합니다.
사용자는 복잡한 코드를 직접 작성하지 않고도 AI 시스템의 실행 흐름을 직관적으로 설계하고 관리할 수 있습니다.


## 2. 추진 배경
최근 LLM 기반 서비스는 단순한 질의 응답을 넘어 RAG, 멀티 에이전트 협업, 조건 분기, 비동기 실행 등 복잡한 워크플로우 구조를 요구하고 있습니다.
그러나 이러한 시스템은 구현 난이도가 높고 실행 흐름을 파악하기 어렵고 디버깅 및 관찰이 제한적이라는 한계가 있습니다.
본 프로젝트는 이러한 문제를 해결하기 위해 “설계한 워크플로우를 그대로 실행하고 관찰할 수 있는 환경”을 제공하고자 합니다.


## 3. 프로젝트 목표
### 기능적 목표
* 시각적 노드 에디터를 통해 AI 워크플로우를 설계
* 설계된 그래프를 실제 AI 실행 엔진으로 변환 및 실행
* 실행 중 각 노드의 상태와 결과를 실시간으로 확인

### 기술적 목표
* 비동기 기반 워크플로우 실행 엔진 구현
* 실행 상태 저장 및 복구 구조 설계
* 스트리밍 응답 및 실시간 상태 동기화


## 4. 핵심 기능
1. 시각적 워크플로우 에디터
* 노드와 연결선 기반의 실행 흐름 설계
* 노드별 설정 및 데이터 흐름 정의

2. RAG 파이프라인 구성
* 문서 로딩, 분할, 임베딩, 검색 단계 분리
* 각 단계의 처리 과정 시각화

3. 멀티 에이전트 워크플로우
* 조건 분기, 반복, 병렬 실행 지원
* 단계별 실행 제어

4. 실시간 실행 관찰
* 실행 중인 노드 하이라이트
* 단계별 입력·출력 및 실행 시간 확인


## 5. 시스템 구성
* 프론트엔드: 시각적 노드 에디터 및 실행 상태 시각화 UI
* 백엔드: 워크플로우 실행 엔진, RAG 처리, 비동기 제어
* 인프라: Vector Database, 상태 저장소, 실시간 통신 채널


## 6. 기대 효과
본 프로젝트를 통해 AI 워크플로우를 구조적으로 설계하고 설명할 수 있는 역량을 증명하고 LLM 활용을 넘어 AI 시스템 인프라에 대한 이해도를 보여줄 수 있으며 프론트엔드, 백엔드, AI를 아우르는 고난도 통합 프로젝트 경험을 확보할 수 있습니다.


## 7. 결론
본 프로젝트는 AI 모델을 사용하는 애플리케이션이 아니라, AI 워크플로우를 설계·실행·관찰하는 플랫폼을 구축하는 것을 목표로 합니다.
