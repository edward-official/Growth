## 키워드 정리
#### MCP(Model Context Protocol)
LLM(모델)이 외부 도구·데이터를 “표준 방식”으로 쓰게 해주는 규약. (LLM이 일관된 인터페이스로 호스트 앱의 파일 시스템, 데이터 베이스, API, 툴 접근)
#### RAG(Retrieval-Augmented Generation)
LLM이 답변을 생성하기 전에 외부 데이터 소스에서 관련 정보를 검색(Retrieval)하고, 그 결과를 컨텍스트로 포함해 응답을 생성하는 방식.
#### 스키마 추출
비정형 텍스트에서 사전에 정의된 필드 구조(Schema)에 맞춰 데이터를 추출해 구조화된 형식(JSON 등)으로 변환하는 과정을 말합니다.
#### 프롬프트 엔지니어링
LLM이 의도한 방식으로 추론·출력하도록 입력 프롬프트(지시, 제약, 예시, 출력 형식)를 설계하는 기술을 말합니다.
#### 샌드박스(Sandbox)
신뢰할 수 없는 코드나 AI가 생성한 작업을 시스템과 격리된 안전한 환경에서 실행하기 위한 개념.
#### 도커(Docker)
애플리케이션과 실행 환경을 격리된 컨테이너로 묶어 실행함으로써 샌드박스를 구현하는 대표적인 기술.
#### vector DB
일반적인 데이터 베이스와 다르게 사람의 기억처럼 의미를 가지고 결과 찾기를 가능하게 해주는 데이터 베이스 ("아이폰 만든 회사" -> "애플" 등)
