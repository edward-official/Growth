## Input / Output (입출력)

### 역할
워크플로우의 시작과 끝을 담당합니다. 사용자의 입력을 받아들이고, 워크플로우의 최종 처리 결과를 사용자에게 적절한 형태로 반환합니다.

### 상세 설명
- **Input (Start)**: 사용자의 질문이나 초기 데이터를 워크플로우의 변수로 주입합니다. 선택적으로 필수 입력값을 정의할 수 있습니다.
- **Output (End/Answer)**: 처리된 데이터를 텍스트, JSON, 스트리밍 등 다양한 포맷으로 변환하여 사용자에게 응답합니다.


## LLM (대형 언어 모델)

### 역할
가장 핵심적인 인지/생성 능력을 제공합니다. 텍스트 생성, 추론, 요약, 번역 등의 작업을 수행합니다.

### LLM 응답 유형 (Response Types)
LLM 노드는 설정에 따라 다음 세 가지 방식으로 작동합니다.

- **자연어 응답 생성 (Natural Language Generation)**: 단순히 프롬프트를 입력받아 자연어 응답을 생성합니다.
- **함수 호출 파라미터 생성 (Function Call Parameter Generation)**: **LLM(뇌)**이 **Tool(손발)**을 사용하기 위한 기능입니다. LLM은 직접 외부 세상과 소통할 수 없으므로, "날씨 검색 도구를 서울 기준으로 실행해줘"와 같은 **명령(JSON 데이터)**을 생성합니다. 즉, 실제 행동을 하는 것은 아니며 "행동을 결심하고 명령서를 작성하는 단계"입니다.
- **구조화된 데이터 포맷 생성 (Structured Data Format Generation)**: 결과값을 JSON 등 정형 데이터 포맷으로 강제하여 시스템이 파싱하기 쉽게 만듭니다.


## Prompt (프롬프트)

### 역할
LLM을 호출하기 위해 전달할 **프롬프트(명령어, 지시문)**를 구성합니다. 다양한 변수와 템플릿을 조합하여 LLM에게 보낼 최종 질문 내용을 만듭니다.

### 상세 설명
- **템플릿 작성 및 변수 연결**: `{{variable}}` 문법으로 빈칸이 있는 문장(**템플릿**)을 만들고, 이 빈칸에 실제 데이터(**변수**)를 채워 넣어 완성된 프롬프트를 생성합니다.
- **프롬프트 버전 관리**: 프롬프트를 수정할 때마다 새로운 버전으로 저장합니다. 여러 가지 문구로 실험해보며 LLM이 가장 잘 알아듣는 최적의 지시문을 찾거나, 문제가 생겼을 때 이전 버전으로 되돌릴 수 있습니다.


## Router (라우터 / 분기)

### 역할
**조건문(If-Else)**의 확장판입니다. 워크플로우의 실행 경로를 결정하는 **'표지판'** 역할을 합니다. 단순한 수치 비교뿐만 아니라 LLM의 판단을 통해 경로를 나눌 수도 있습니다.

### 상세 설명
- **룰 기반 (Rule-based)**: "값이 10보다 크면 A경로, 아니면 B경로"와 같이 명확한 논리 조건에 따라 분기합니다.
- **LLM 분기 (Classifier)**: LLM이 텍스트의 의도를 파악하여 경로를 결정합니다. (예: 질문이 "환불" 관련이면 A, "배송" 관련이면 B)


## Retriever (검색 / RAG)

### 역할
LLM이 학습하지 않은 외부 지식이나 사내 문서 등을 검색하여 답변에 활용할 수 있게 합니다 (Retrieval-Augmented Generation).

### 상세 설명
사용자의 질문을 벡터로 변환(Embedding)하고, 벡터 데이터베이스에서 가장 유사한 문서를 찾아와 LLM에게 문맥(Context)으로 제공합니다.


## Tool (도구)

### 역할
LLM이 외부 세계와 상호작용할 수 있는 능력을 부여합니다. 단순한 텍스트 생성을 넘어 실질적인 작업을 수행합니다. **MCP (Model Context Protocol)**와 같은 표준 프로토콜을 사용하여 다양한 외부 도구와 쉽게 연결될 수 있습니다.

### 상세 설명
- **HTTP**: 외부 API 서버에 요청을 보냅니다 (예: 날씨 정보 조회, 슬랙 메시지 전송).
- **DB/파일**: 데이터베이스를 조회하거나 파일을 읽고 씁니다.


## Code (코드 실행)

### 역할
복잡한 수치 계산, 데이터 가공 등 LLM이 취약한 논리적인 처리를 프로그래밍 코드로 직접 수행합니다.

### 상세 설명
보안이 유지되는 **샌드박스 환경**에서 Python이나 JavaScript 코드를 실행하고, 그 결과를 워크플로우 변수로 반환합니다.
- **보안(Security)**: 악의적인 코드 실행(예: `rm -rf /`, 무한 루프)이나 민감한 시스템 파일 접근(예: 환경 변수 탈취)을 원천적으로 차단하여 서버를 보호합니다.
- **안정성(Stability)**: 사용자 코드가 과도한 메모리를 사용하거나 충돌하더라도, 전체 시스템에 영향을 주지 않고 해당 노드만 안전하게 종료시킬 수 있습니다.


## Loop (반복)

### 역할
데이터 리스트를 순회하며 동일한 작업을 반복 수행합니다 (프로그래밍의 For-each 문).

### 상세 설명
- **Loop**: 리스트 형태의 데이터를 받아 각 항목에 대해 설정된 하위 워크플로우를 반복적으로 실행하고, 결과를 모아서 반환합니다.


## Memory (메모리)

### 역할
멀티턴(Multi-turn) 대화에서 이전 대화의 맥락을 기억하여 연속성 있는 대화를 가능하게 합니다. (플랫폼에 따라 별도 노드로 지원하거나, 내부 변수로 처리하는 등 지원 방식이 다를 수 있습니다.)

### 상세 설명
- **대화 메모리 (Conversation)**: 최근 N개의 대화 주고받음을 그대로 저장했다가 프롬프트에 포함합니다.
- **요약 메모리 (Summary)**: 대화가 길어질 경우, 과거 내용을 요약하여 핵심 정보만 유지함으로써 토큰 비용을 절약합니다.


## Guardrail (가드레일 / 안전장치)

### 역할
AI의 입출력을 모니터링하여 보안 위협, 개인정보 유출, 유해한 콘텐츠 생성을 방지합니다. (플랫폼에 따라 별도 노드로 지원하거나, 내부 정책으로 처리하는 등 지원 방식이 다를 수 있습니다.)

### 상세 설명
- **PII 마스킹**: 전화번호, 주민번호 등 개인식별정보가 포함되면 자동으로 가리거나 차단합니다.
- **정책 필터링**: 폭력적, 선정적인 내용이나 시스템 프롬프트 유출 시도를 탐지하고 차단합니다.
