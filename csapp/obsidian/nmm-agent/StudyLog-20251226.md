# React useRef 정리

## 개요
useRef는 React에서 **값이나 DOM 요소를 컴포넌트의 생명주기 동안 유지하되, 해당 값의 변경이 렌더링을 유발하지 않도록 하기 위해 제공되는 훅**입니다.  
주로 렌더링과 직접적인 연관이 없는 정보나 DOM 참조를 관리할 때 사용됩니다.

---

## useRef의 주요 특징
- useRef로 생성된 값은 변경되더라도 컴포넌트를 다시 렌더링하지 않습니다.
- 컴포넌트가 여러 번 렌더링되더라도 동일한 ref 객체가 유지됩니다.
- 실제 값은 `ref.current` 프로퍼티에 저장됩니다.
- useRef는 상태(state) 관리보다는 **참조(reference) 관리 목적**에 적합합니다.

---

## 사용 목적 및 활용 사례

### 1. 렌더링과 무관한 값 관리
화면에 표시할 필요는 없지만,  
컴포넌트가 유지되는 동안 지속적으로 관리되어야 하는 값에 useRef를 활용합니다.

대표적인 활용 사례는 다음과 같습니다.
- 비동기 요청의 중복 실행을 방지하기 위한 플래그
- 타이머 또는 인터벌 ID 관리
- 이전 상태 값 보관
- 임시 캐시 데이터 저장

이러한 값들을 useState로 관리할 경우,  
불필요한 리렌더링이 발생할 수 있으므로 useRef가 더 적합합니다.

---

### 2. DOM 요소 직접 제어
useRef는 React의 선언적 UI 흐름 내에서  
DOM 요소를 직접 제어해야 하는 상황에서도 사용됩니다.

주요 활용 예시는 다음과 같습니다.
- 입력 요소에 대한 포커스 제어
- 스크롤 위치 확인 및 조작
- 요소의 크기 또는 위치 측정
- 모달 외부 클릭 감지 처리

이와 같은 경우, useRef를 통해 DOM 요소에 안전하게 접근할 수 있습니다.

---

## useState와의 차이점
useState는 값의 변경이 화면 렌더링에 반영되어야 하는 경우에 사용합니다.  
반면, useRef는 값의 변경이 화면에 영향을 주지 않아야 하는 경우에 사용합니다.

정리하면 다음과 같습니다.
- UI 상태 관리 → useState
- 렌더링과 무관한 로직 보조 정보 또는 DOM 참조 → useRef

---

## 실제 적용 사례 (CreateAppModal)
CreateAppModal 컴포넌트에서는 다음과 같은 이유로 useRef가 활용됩니다.
- 생성 요청이 이미 진행 중인지 여부를 기억하여 중복 요청을 방지하기 위함
- 모달 요소에 직접 접근하여 포커스 제어 및 외부 클릭 감지를 처리하기 위함

이러한 정보는 화면 렌더링과 직접적인 관련이 없기 때문에  
useRef를 사용하는 것이 적절한 선택입니다.

---

## 결론
> **useRef는 화면에 영향을 주지 않는 값이나 DOM 요소를 컴포넌트 생명주기 동안 안정적으로 유지해야 할 때 사용하는 React 훅입니다.**
